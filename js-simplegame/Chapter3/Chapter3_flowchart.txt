1. word-relay
    # 순서도 그리기
        1. 게임에 몇 명이 참가할지를 선택한다.
        2. 참가자 순서를 정한다(편의상 숫자로 정한다.)
        3. 첫 번째 사람이 어떤 단어를 말한다.
        4. 다음 사람이 어떤 단어를 말한다.
        5. 절차 4에서 말한 단어가 올바른지 판단한다.
        6. 올바르다면 그 다음 사람이 어떤 단어를 말한다.
        7. 올바르지 않다면 틀렸다고 표시한다.
        8. 게임을 계속 진행한다.

2. 쿵쿵따
    # 순서도 그리기
        1. 게임에 몇 명이 참가할지를 선택한다.
        2. prompt에서 확인과 취소를 통해 게임 실행 여부를 선택한다.
        3. 참가자 순서를 정한다(편의상 숫자로 정한다.)
        4. 첫 번째 사람이 어떤 단어를 말한다.
        5. 어떤 단어의 글자 수가 3인지 확인한다.
        6. 다음 사람이 어떤 단어를 말한다.
        7. 절차 4,5를 통해 말한 단어가 올바른지 판단한다.
        8. 올바르다면 그 다음 사람이 어떤 단어를 말한다.
        9. 올바르지 않다면 틀렸다고 표시한다.
        10. 게임을 계속 진행한다.


* 웹 브라우저 대화 상자
    - prompt: 대화 상자에 사용자가 입렫한 메시지가 문자열 형태로 전달되고, 입력하지 않고 취소를 누르면 null이 전달됩니다.

    - alert: 단순한 알림창으로, 호출 시 확인을 누르기 전까지 다음 스크립트 실행이 중단됩니다.

    - cofirm: 사용자에게 확인을 받을 때 사용합니다. 사용자가 확인을 누르면 'true'를 취소를 누르면 false가 전달됩니다.

* HTML 태그 선택
    - document.querySelector('선택자'): 하나의 태그 선택
    - document.querySelectorAll('선택자'): 여러 개의 태그 선택
    - 선택자: #아이디 or .클래스

* 태그에 이벤트 달기
    - addEventListener: 태그에 이벤트 리스너를 통해 이벤트가 발생할 때마다 리스너 함수가 실행되도록 함.
    태그.addEventListener('이벤트 이름', 리스너 함수)
